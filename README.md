# RAG智能客服系统

## 项目简介

本项目是一个基于RAG（检索增强生成）技术的智能客服系统，使用Python实现，前端采用Streamlit构建，后端使用Chroma向量库和通义千问大模型，为用户提供基于知识库的智能问答服务。

## 技术栈

- **前端**：Streamlit
- **后端**：Python
- **向量存储**：Chroma
- **嵌入模型**：DashScope Embeddings
- **大语言模型**：通义千问（qwen3-max）
- **知识处理**：LangChain

## 项目结构

```
RAG智能客服/

└── vectore_store.py    # 向量存储服务
```

## 核心功能

### 1. 知识库管理

- **文件上传**：支持上传CSV、TXT格式文件到知识库
- **内容去重**：通过MD5哈希值检测避免重复内容
- **文本分割**：自动将长文本分割成适合向量存储的小块
- **向量存储**：使用Chroma向量库存储文本嵌入
- **知识库操作**：支持根据内容、文件名删除知识库数据，以及清空整个知识库

### 2. 智能问答

- **向量检索**：根据用户问题从知识库中检索相关文档
- **上下文增强**：将检索到的文档作为上下文提供给大模型
- **对话历史**：记录和管理用户与系统的对话历史
- **流式输出**：支持大模型的流式输出，提升用户体验

### 3. 系统架构

- **前端**：使用Streamlit构建直观的用户界面
- **后端**：基于LangChain构建RAG流程
- **模型**：集成通义千问大模型进行智能回答
- **存储**：使用Chroma向量库和本地文件系统存储数据

## 工作流程

1. **知识库构建**：
   - 用户上传文件到系统
   - 系统计算文件内容的MD5值，检查是否已存在
   - 对文件内容进行分割处理
   - 将分割后的文本转换为向量并存储到Chroma向量库
   - 记录文件的MD5值到md5.txt

2. **智能问答**：
   - 用户输入问题
   - 系统从向量库中检索与问题相关的文档
   - 将检索到的文档与问题一起构建提示词
   - 调用通义千问大模型生成回答
   - 记录对话历史并返回回答给用户

## 配置说明

项目配置文件为`config.py`，包含以下配置项：

- **chrom**：向量库配置
- **text_splitter**：文本分割配置
- **retrieval**：检索配置
- **model**：模型配置
- **vector_store**：向量存储配置
- **session**：会话配置

## 使用方法

### 1. 环境准备

```bash
# 安装依赖
pip install -r requirements.txt
```

### 2. 运行应用

#### 启动智能问答应用

```bash
streamlit run app_qu.py
```

#### 启动文件上传应用

```bash
streamlit run app_upload_file.py
```

### 3. 使用流程

1. **上传文件**：
   - 访问文件上传应用
   - 选择要上传的文件（支持CSV、TXT格式）
   - 点击上传按钮，系统会自动处理文件并添加到知识库

2. **智能问答**：
   - 访问智能问答应用
   - 在聊天输入框中输入问题
   - 系统会基于知识库内容生成回答
   - 对话历史会被自动记录

## 注意事项

1. 本项目使用了通义千问大模型和DashScope Embeddings，需要配置相应的API密钥
2. 向量库数据存储在`data/chroma_db`目录中，对话历史存储在`history`目录中
3. 上传的文件内容会被分割成小块并转换为向量存储，因此文件大小应合理控制
4. 系统会自动检测重复内容，避免知识库中出现重复数据
